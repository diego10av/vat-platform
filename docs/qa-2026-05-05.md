## Bugs identified during QA

### BUG-001 — Login form asked for username (FIXED in this session)

`src/app/login/page.tsx` showed two inputs (Username + Password)
inherited from the multi-user model. In single-user (post-reset)
only the password is needed. **Fixed**: form simplified to a single
password input with autofocus, fetch sends only `{ password }`.

### BUG-002 — "Open 2027" pill overlapped with the /tax-ops title

When the viewport was narrow, the "📅 Open 2027" pill in the top-
right of the `/tax-ops` PageHeader fell on top of the "Ops" word in
the h1. Visible in `Tax-Ops` matching Diego's quote ("pages that
open and overlap with the previous one").

**Hypothesis**: `<PageHeader>` did not apply `flex-wrap` or the
floating button used `position: absolute` without a clear z-index.
Reproduce below `<1300px` and review `src/components/ui/PageHeader.tsx`
+ the wrapper in `src/app/tax-ops/page.tsx`.

**Severity**: medium. Disrupts title reading but does not block.

### BUG-003 — Sidebar still carries `roles` arrays + `filterForRole`
(cosmetic)

`src/components/app-shell/Sidebar.tsx` still defines `Role` type +
`roles?: ['admin', 'reviewer']` annotations on `NavItem` / `NavGroup`
+ a `filterForRole` function. In single-user `role` is hardcoded to
`'admin'` (`const role: Role = 'admin'`), so `filterForRole` is the
identity. The code works but carries 80+ lines that no longer mean
anything.

**Severity**: low. Functional, just cosmetic cleanup.

### BUG-004 — Modal backdrops too transparent (FIXED in Phase 7.3)

Diego: "many things are broken in this style" — sent a screenshot
of the Switch VAT subtype modal where the contents looked like they
overlapped with the table behind. Root cause was the backdrop being
too transparent (`bg-ink/60` on the primitive, `bg-black/30-40` on
the half-dozen modals that roll their own overlay). Through that
thin scrim the table stayed half-readable, creating the optical
illusion that the modal content floated over the page rather than
separating modal-mode cleanly.

**Fixed**: primitive `Modal` → `bg-ink/75 backdrop-blur-[6px]`,
primitive `Drawer` → `bg-ink/50 backdrop-blur-[3px]`, plus eight
roll-your-own modals (BulkEditDrawer, GlobalSearch, FeedbackWidget,
FilingEditDrawer, contacts switch-firm modal, ExcelImportModal,
BulkEditModal, AttachmentsModal) normalised to the same standard.

## How to work the list

Each bug you find becomes an entry in `docs/TODO.md` under "This
week". Each fix:

1. Reproduce the bug (one line: route + action).
2. Hypothesise the cause (one line).
3. Fix on a branch or directly on main if trivial.
4. One commit, message `fix(area): <what>`.
5. Verify build green before push.

For visual issues (overflow, alignment) use preview tools:
- `preview_screenshot` for before / after.
- `preview_inspect` with the specific CSS property to verify.
